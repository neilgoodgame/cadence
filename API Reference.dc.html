A REST API over the same data model that powers the app — activities and their 1 Hz streams, designed workouts, scheduling, athlete thresholds & zones, coaching, and gear. JSON over HTTPS, OAuth 2.0, predictable resource URLs.

Authentication

OAuth 2.0 authorization code flow. Send the access token as a bearer header on every request.

Authorization: Bearer cad_at_9f2a7c…
Pagination

List endpoints are cursor-paginated. Pass cursor from next_cursor and an optional limit (max 200).

GET /v1/activities?limit=50&cursor=act_8KQ2
Query language

The q parameter on GET /v1/activities accepts the same CQL used by Activities search — field comparisons, AND/OR, and ordering.

q=sport = run AND tag ~ race AND distance > 42.2km&sort=-tss
